Создание проекта на  Github

Чтобы опубликовать лендинг или другой статический сайт на Netlify, не обязательно использовать git. Можно просто загрузить сайт  в виде zip-архива.

Но такой способ даст нам только быстрый и бесплатный хостинг с автоматическим ssl. Что в принципе, уже неплохо. 

Однако, Netlify может предложить нам гораздо больше интересного функционала, если его использовать в связке с git:

  • автоматическое развертывание дополнительных вариантов лендинга с различными офферами под разные объявления в директе, всего-лишь создавая новые ветки;
  • создание AB-тестов из веток;
  • оптимизируя код, можно сразу контролировать, как на эти изменения будет реагировать PageSpeed, не боясь что-то "сломать" на работающей версии

Почему именно GitHub, а не бесплатный закрытый аккаунт Bitbucket:

Наш лендинг — статический сайт, состоящий только из html, css и js. А эта часть любого сайта всегда доступна с помощью средств разработчика браузера. Поэтому прятать в bitbucket тут нечего: не то чтобы у нас нет секретного бэкенда — у нас нет бэкенда в принципе. В этом и есть главная фича статических сайтов. Поэтому они быстрые и безопасные — сломать их не возможно.

С другой стороны, если к лендингу потребуется прикрутить визуальный редактор (Netlify CMS), то бета-версия работает пока только c GitHub. 

К тому же открытый репозиторий делает из нашего лендинга open source проект и мы получаем от нетлифи больше полезного функционала.

Тем, кто беспокоится о данных пользователей при заполнении формы:

пользовательская информация из форм обрабатывается отдельно вне сайта и c git никак не связана в принципе

Итак, сначала я создаю новый проект на GitHub, инициализирую его и только после этого перехожу к Netlify.

Как подключить к Netlify домен третьего уровня

Порядок действий зависит от того, где располагается основной домен. 

  1. Если основной домен расположен где-то на другом хостинге, то сначала необходимо добавить новую запись CNAME с именем поддомена на том сервисе, где базируется основной сайт. После этого можно привязать этот домен к  сайту на Netlify, как указано выше.

  2. Если основной домен уже привязан к Netlify, то можно точно также добавить к основному сайту CNAME-запись и также выполнить все манипуляции по добавлению нового проекта, как описано выше. А можно воспользоваться функцией "новый поддомен из git-ветки". И получить автоматческое развертывание, ssl и т. д., просто создав новую ветку на git.

Еще о преимуществах Netlify для публикации лендингов

Кроме того, что мы получаем бесплатный, высокоскоростной хостинг с автоматически возобновляемым ssl-сертификатом, у сервиса есть еще ряд полезных для лендингов функций:

Подключение форм

Для статических сайтов формы могут стать головной болью, ведь обычно для работы с формами необходим бэкенд, который на статике отсутствует. 

На Netlify подключение форм уже предусмотрено. При этом, на мой взгляд, с помощью "родного" функционала Netlify заставить работать формы проще, быстрее и удобнее, чем это реализовано в других сервисах. Защита от ботов и встроенная невидимая капча тоже присутствует.

Данные по отправленным заявкам можно увидеть в разделе "Forms". В этом же разделе сразу подключаю уведомления на почту того, кто эти заявки будет обрабатывать и интегрирую в канал slack.

Тем, кто публикует свой статический сайт не на Netlify, рекомендую посмотреть в сторону formester . Сервис можно использовать бесплатно, если поступает до 100 заявок в месяц. При большем количестве заявок, использование сервиса обойдется 13$ в месяц. Но сервис потребует чуть больше манипуляций по настройке:  придется самостоятельно подключить рекапчу, указать страницу успешной отправки формы, добавить интеграции по заполненным формам через zapier.

Возможности для проведения АБ-тестов

Чтобы проводить тестирование 2-х разных версий лендинга, нам необходимо разместить вторую версию на подветке в git. Дальше идем в раздел "Split testing" , где настраиваем и запускаем процесс тестирования:

Можно добавить не одну, а несколько альтернативных версий, аналогично добавляя дополнительные ветки. После того, как процесс тестирования запущен, посетителям будет отображаться рандомно одна из включенных в тестирование версий сайта. То есть все эти версии будут открываться по адресу yoursitename.ru

Изменение текстов офферов (и не только текстов) под различные объявления

Если вы хотите, чтобы под каждое рекламное объявление показывалась не рандомная, а определенная версия, то просто создаем поддомен из ветки, без включения в сплит-тест. И уже в директе указываем ссылку на нужную версию. 

Важно: имя поддомена из ветки будет иметь вид <имя ветки>.yoursitename.ru Елли ветка для сплит-тестирования, то ее название не важно. В случае с изменением страниц под конкретные объявления, имя подветки все-таки должно быть презентабельным

Подключение сторонних сервисов (например, аналитики), минуя код

Добавить код, например, Яндекс Метрики, Гугл аналитики, рекапчи, или чего-то еще можно и не изменяя html-файл. Все это можно сделать, в разделе "Settings":

Если ориентироваться на Google PageSpeed, установка кода в html руками все-таки предпочтительнее. Сравнивала оба способа. К примеру, код Яндекс Метрики, установленный через постпроцессинг "съедает" 5 баллов в PageSpeed.
